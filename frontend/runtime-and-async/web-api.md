# Event Loop

## 핵심 질문
왜 비동기 코드는 작성 순서대로 실행되지 않는가?

---

## 헷갈렸던 지점
- `setTimeout`이 먼저 실행될 것이라고 생각했다

비동기 코드는  
작성된 순서대로 큐에 들어가고,  
그 순서대로 실행될 것이라고 막연히 기대하고 있었다.

---

## 구조적 이해
- 콜 스택: 현재 실행 중인 자바스크립트 코드
- 태스크 큐 / 마이크로태스크 큐: 비동기 작업의 대기 공간
- 이벤트 루프: 콜 스택이 비었을 때,  
  **어떤 큐의 작업을 언제 스택으로 올릴지 결정하는 역할**

이 구조 때문에  
작성 순서와 실행 순서는 항상 일치하지 않는다.

---

## 실무 기준
- 비동기 로직은 실행 순서를 가정하지 않는다
- 순서가 아니라 **의존성과 완료 조건**을 기준으로 설계한다  
  (어떤 작업이 끝나야 다음 작업이 의미를 가지는지 먼저 정의한다)

# Runtime Model

## 핵심 질문
자바스크립트 코드는 어디에서 실행되는가?

---

## 내가 헷갈렸던 지점
- JS가 모든 작업을 직접 처리한다고 생각했다
- 브라우저와 JS 엔진의 역할을 명확히 구분하지 못했다

이로 인해 비동기 문제를  
항상 “자바스크립트 코드의 문제”로만 바라보고 있었다.

---

## 구조적으로 이해한 내용
- JS 엔진: 콜 스택을 기반으로 자바스크립트 코드를 실행
- 브라우저: 타이머, 네트워크 요청 등 Web API 제공
- 런타임: JS 엔진과 이를 둘러싼 **호스트 환경의 조합**

---

## 실무 기준
- 비동기 문제가 발생했을 때,  
  먼저 자바스크립트 코드가 아니라  
  **실행 환경(브라우저, Web API, 런타임 구조)**을 의심한다

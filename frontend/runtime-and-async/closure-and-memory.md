# Closure and Memory

## 핵심 질문
이미 끝난 함수의 변수는 왜 살아 있는가?

---

## 헷갈렸던 지점
- 함수 실행이 끝나면 관련 변수도 함께 사라질 것이라고 생각했다

함수의 실행이 끝났다는 사실과  
메모리에서 해제된다는 사실을 동일하게 인식하고 있었다.

---

## 구조적 이해
- 실행 컨텍스트의 종료는  
  해당 함수의 실행이 끝났음을 의미할 뿐, 메모리 해제를 보장하지 않는다
- 변수가 다른 곳에서 **참조되고 있다면**,  
  가비지 컬렉션의 대상이 되지 않고 메모리에 유지된다

이로 인해 함수 실행이 끝난 이후에도  
클로저를 통해 변수에 접근할 수 있다.

---

## 실무 기준
- 불필요한 참조는 의도치 않은 메모리 유지를 초래한다
- 클로저를 사용할 때는  
  **변수의 수명과 참조 관계**를 함께 고려한다

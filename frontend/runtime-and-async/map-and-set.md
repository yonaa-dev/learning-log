# Map and Set

## 핵심 질문
객체 대신 Map/Set을 써야 하는 순간은 언제인가?

---

## 헷갈렸던 지점
- 키-값 구조라면 객체로도 충분하다고 생각했다

객체와 Map/Set의 차이를  
성능이나 문법 차이 정도로만 인식하고 있었다.

---

## 구조적 이해
- Map / Set은 단순한 저장소가 아니라 **컬렉션**을 표현하기 위한 자료구조
- 키의 타입 제약이 없고, 삽입 순서가 보장된다
- 데이터의 성격과 사용 의도를 코드 구조로 드러낼 수 있다

---

## 실무 기준
- 단순 속성 묶음이 아니라  
  **“여러 항목을 관리한다”는 컬렉션 의도**가 있을 때 Map/Set을 사용한다

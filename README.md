# Learning Log

이 저장소는 개발자로 일하며 느꼈던 한계와 고민을 바탕으로,  
기초 개념부터 다시 정리하며 **학습 과정과 사고의 변화를 기록하기 위한 개인 학습 로그**입니다.

단순한 강의나 책 정리가 아니라,  
학습 과정에서 반복적으로 마주친 **개념적 혼란과 설계 질문**을 기록하는 것을 목표로 합니다.

“무엇을 배웠는지”보다  
**“왜 헷갈렸는지, 무엇을 다시 정의했는지, 사고가 어떻게 바뀌었는지”** 를 남깁니다.

---

## Why this repository exists

프론트엔드 개발자로 일하며  
기술은 사용했지만, 개념과 설계에 대한 이해가 충분하지 않았다는 한계를 느꼈습니다.

이 저장소는  
**“어디까지 알고 있었는지”보다  
“무엇을 이해하지 못한 채 지나왔는지”를 다시 점검하기 위한 기록**입니다.

성장 중인 나 자신을 위한 기록이자,  
개발자로서의 방향성과 사고 기준을 다시 정리하기 위한 공간입니다.

---

## What I document here

이 저장소에는 다음과 같은 내용을 기록합니다.

- 학습 중 중요하다고 느낀 개념과 포인트
- 이해가 어려웠던 부분과 그에 대한 나만의 해석
- 실무 또는 프로젝트에 어떻게 연결할 수 있을지에 대한 고민
- 스터디 발표, 개발 행사, 북토크, 네트워킹 이후의 생각 정리

---

## Writing principles

이 저장소는 완성된 정답 노트가 아니라,  
**생각이 정리되어 가는 과정을 기록하는 공간**입니다.

- 강의 이름이나 책 이름으로 폴더를 만들지 않는다
- 기술 목록이 아닌 **질문과 판단 기준** 중심으로 기록한다
- 정답보다 **사고 과정**을 남긴다
- 학습 기록은 언제든 수정·확장될 수 있다
- 나중에 면접에서 말로 설명할 수 있는 수준을 목표로 한다

---

## Repository Structure

- `frontend/`
  - 프론트엔드 개발과 관련된 개념을 **설계 질문 단위**로 정리
  - 개념 → 헷갈린 지점 → 나만의 해석 → 실무 연결 순으로 기록

- `weekly/`
  - 주 단위 학습 회고
  - 이해한 것 / 막혔던 것 / 다음 주 계획 포함

- `sources.md`
  - 이 저장소에서 참고한 강의, 도서, 문서 목록

---

## Goal

이 저장소의 목표는 단순한 지식 축적이 아니라,  
**문제를 어떻게 이해하고, 어떤 기준으로 판단했는지를 설명할 수 있는 개발자로 성장하는 것**입니다.

각 기록은  
- 개념을 어떻게 재정의했는지  
- 실무 또는 프로젝트에 어떻게 연결할 수 있는지  
- 이후 어떤 질문으로 확장되었는지  

를 남기는 데 초점을 둡니다.

다시 기초부터 쌓아가는 과정을  
**스스로 증명하기 위한 학습 기록**입니다.
